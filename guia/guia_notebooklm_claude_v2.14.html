1

Instalar o notebooklm-py no seu computador

O login no NotebookLM não funciona dentro do Google Colab — ele precisa de um navegador real para abrir a tela de autenticação do Google. Por isso, o primeiro passo é instalar o notebooklm-py na sua máquina local.

💡 Por que instalar localmente? O processo de login abre um navegador Chrome real para você fazer login no Google. O Colab roda em servidores na nuvem sem interface gráfica, então esse passo precisa ser feito na sua máquina. Depois de gerar o arquivo de credenciais, você o envia para o Colab usar.
1
Instale o Python no seu computador
O Python é necessário para rodar o notebooklm-py. Siga as instruções do seu sistema:
🏟 Windows
  1. Acesse python.org/downloads e clique em Download Python
  2. Abra o instalador baixado
  3. ⚠ MARQUE a caixa "Add Python to PATH" antes de continuar
  4. Clique em Install Now e aguarde finalizar
⚠️ Se esquecer de marcar "Add to PATH", o comando notebooklm nao vai funcionar no terminal. Nesse caso, desinstale e reinstale marcando a opcao.
🍎 Mac
  1. Acesse python.org/downloads e baixe a versao para macOS
  2. Abra o arquivo .pkg e siga o assistente — o PATH e configurado automaticamente
Para confirmar que funcionou, abra o terminal e execute:
terminal
python --version
Deve aparecer algo como Python 3.12.0. Se aparecer erro, repita a instalacao marcando "Add to PATH".
2
Instale o notebooklm-py localmente
No terminal do seu computador, execute:
terminal
pip install notebooklm-py playwright
playwright install chromium
💡 Isso instala o notebooklm-py e o Chromium (navegador que ele usa para fazer login). Pode demorar 2-3 minutos.
2

Fazer login e gerar o arquivo de credenciais

Com o notebooklm-py instalado localmente, agora você vai fazer login no Google. Isso gera um arquivo storage_state.json com suas credenciais — é ele que o Colab vai usar para se autenticar.

1
Execute o login no terminal local
No terminal do seu computador (não no Colab), execute:
terminal
notebooklm login
Uma janela do Chrome vai abrir automaticamente com a tela de login do Google.
2
Faça login com sua conta Google
Na janela que abriu, entre com o e-mail e senha da sua conta Google. Após o login, aguarde chegar na página inicial do NotebookLM. Quando estiver lá, volte ao terminal e pressione Enter para salvar as credenciais.
⚠️ Use a mesma conta Google onde seus notebooks estão salvos.
3
Localize o arquivo gerado
O arquivo storage_state.json fica na pasta oculta .notebooklm do seu usuario:
🏟 Windows:
C:\Users\SeuUsuario\.notebooklm\storage_state.json
No Explorador de Arquivos, cole %USERPROFILE%\.notebooklm na barra de endereco e pressione Enter
🍎 Mac:
~/.notebooklm/storage_state.json
No Finder, pressione Cmd+Shift+G e cole ~/.notebooklm
💡 Confirme executando notebooklm list no terminal. Se seus notebooks aparecerem, o login funcionou!
📌 Guarde esse arquivo! Voce vai precisar dele no Passo 3. Quando as credenciais expirarem, repita o login e pegue o novo arquivo no mesmo local.
🔄 Credenciais expiraram? As credenciais duram algumas semanas. Quando aparecer "Authentication expired" no Studio, basta repetir este passo: execute notebooklm login no terminal local, localize o novo storage_state.json (Passo 2, item 3) e faça upload novamente na célula de setup do Colab.

Configurar o Google Colab

Acesse o Google Colab e crie um notebook

Instale as dependências no Colab
Clique em + Código, cole o conteúdo abaixo na célula criada e execute com Ctrl+Enter ou o botão ▶:

3
Instale as dependências e faça upload das credenciais
Crie uma nova célula e cole o script abaixo. Ao executar, ele instala as dependências e abre um botão de upload — selecione o arquivo storage_state.json gerado no Passo 2:
python
import os
import json
from google.colab import files

print('--- Instalando dependencias ---')
os.system('pip install notebooklm-py flask flask-cors pyngrok requests -q')
# Reinstala para garantir versao completa e funcional
os.system('pip install --force-reinstall notebooklm-py -q')

print('\n--- Autenticacao ---')
print('Faca o upload do seu arquivo storage_state.json:')
uploaded = files.upload()

if 'storage_state.json' in uploaded:
    auth_content = uploaded['storage_state.json'].decode('utf-8')
    try:
        json.loads(auth_content)
        # Salva o arquivo no diretorio esperado pela CLI
        os.makedirs(os.path.expanduser('~/.notebooklm'), exist_ok=True)
        path = os.path.expanduser('~/.notebooklm/storage_state.json')
        with open(path, 'w') as f:
            f.write(auth_content)
        # Define tambem como variavel de ambiente (dupla compatibilidade)
        os.environ['NOTEBOOKLM_AUTH_JSON'] = auth_content
        print('\n✓ Credenciais configuradas com sucesso!')
        print('\n--- Verificando login ---')
        import subprocess
        result = subprocess.run(
            ['notebooklm', 'list'],
            capture_output=True, text=True,
            env=os.environ
        )
        if result.returncode == 0:
            print('✓ Login OK! Seus notebooks:')
            print(result.stdout)
        else:
            print('Erro:', result.stderr)
            print('Verifique se o storage_state.json esta correto e tente novamente.')
    except Exception as e:
        print(f'Erro: {e}')
else:
    print('Erro: arquivo storage_state.json nao encontrado.')
⚠️ Este upload precisa ser feito toda vez que o Colab reiniciar a sessão, pois os arquivos não persistem entre sessões gratuitas.
4

Iniciar o servidor

Este é o passo que conecta tudo. O servidor Flask expõe os comandos notebooklm-py via HTTP. O ngrok cria um endereço público para acessá-lo.

1
Obtenha e configure seu token do ngrok
Acesse dashboard.ngrok.com, crie uma conta gratuita e copie seu token. Depois escolha uma das duas opções para configurá-lo no Colab:
✅ Opção A — Colab Secrets (recomendada)
  1. No Colab, clique no ícone de 🔑 chave na barra lateral esquerda (abaixo dos ícones de arquivos, pesquisa e código)
  2. Clique em "Add new secret"
  3. No campo Name, digite exatamente: NGROK_TOKEN
  4. No campo Value, cole o token copiado do dashboard do ngrok
  5. À direita dos campos, ative o toggle "Notebook access" — sem isso, o código não consegue ler o token
  6. Clique em Save
✓ Pronto. O token fica salvo permanentemente no Colab e nunca aparece visível no código.
⚡ Opção B — Direto no código (rápida)
No código do servidor (próximo passo), localize a linha:
ngrok_token = 'SEU_TOKEN_AQUI'
e substitua SEU_TOKEN_AQUI pelo seu token real.
⚠️ Não compartilhe o notebook com o token visível.
⛔ Se deixar SEU_TOKEN_AQUI no código, o servidor vai travar com erro ERR_NGROK_105. O código agora valida automaticamente e exibe uma mensagem clara caso o token não esteja configurado.
2
Cole e execute o código do servidor
Crie uma nova célula, cole o código abaixo e execute. Deixe essa célula rodando o tempo todo que for usar o sistema.
⚠️ Antes de executar, configure o token do ngrok — veja as duas opções comentadas no código (via Colab Secrets ou direto no código). Se deixar SEU_TOKEN_AQUI, o servidor vai travar com erro de autenticação.
python
from flask import Flask, request, jsonify, send_from_directory
from flask_cors import CORS
from pyngrok import ngrok
import subprocess, requests, os, time

app = Flask(__name__)
CORS(app, origins="*")

AUTH_FILE = '/content/storage_state.json'
CONTENT_DIR = '/content/'

# Opção: carregar credenciais do Google Drive (sem reupload manual)
# import gdown
# gdown.download(id='SEU_ID_NO_DRIVE', output=AUTH_FILE, quiet=True)

if not os.path.exists(AUTH_FILE):
    raise FileNotFoundError(f'Credenciais não encontradas em {AUTH_FILE}.')

def run_notebooklm(cmd, retries=3, delay=30):
    with open(AUTH_FILE, 'r') as f:
        auth_content = f.read().strip()
    env = os.environ.copy()
    env['NOTEBOOKLM_AUTH_JSON'] = auth_content
    for attempt in range(retries):
        result = subprocess.run(
            f'notebooklm {cmd}',
            shell=True, capture_output=True,
            text=True, timeout=300, env=env
        )
        # Retry automático em rate limit do Google
        if result.returncode != 0 and 'GENERATION_FAILED' in (result.stdout + result.stderr):
            if attempt < retries - 1:
                time.sleep(delay * (attempt + 1))
                continue
        return result
    return result

@app.route('/execute', methods=['POST'])
def execute():
    cmd = request.json.get('command', '')
    try:
        result = run_notebooklm(cmd)
        return jsonify({'stdout': result.stdout, 'stderr': result.stderr, 'returncode': result.returncode})
    except Exception as e:
        return jsonify({'error': str(e)}), 500

@app.route('/health', methods=['GET'])
def health():
    return jsonify({'status': 'ok'})

@app.route('/download/<path:filename>', methods=['GET'])
def download(filename):
    try:
        return send_from_directory(CONTENT_DIR, filename, as_attachment=True)
    except FileNotFoundError:
        return jsonify({'error': 'File not found'}), 404

# v2.3: salvar histórico de comandos do Studio no Colab
@app.route('/save-history', methods=['POST'])
def save_history():
    import json, datetime
    data = request.json.get('history', [])
    path = '/content/nlm_history.json'
    with open(path, 'w') as f:
        json.dump({'version':'2.4','synced':datetime.datetime.now().isoformat(),'commands':data}, f, indent=2)
    return jsonify({'ok': True, 'path': path, 'count': len(data)})

# PASSO OBRIGATÓRIO: configure seu token do ngrok
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# OPÇÃO A (recomendada — token não aparece no código):
#   1. No Colab, clique em 🔑 (ícone de chave) na barra lateral esquerda
#   2. Ou acesse: Runtime > Manage Secrets
#   3. Clique em "Add new secret"
#   4. Name: NGROK_TOKEN  |  Value: seu_token_aqui
#   5. Ative o toggle "Notebook access"
#
# OPÇÃO B (simples — cole o token diretamente):
#   Substitua SEU_TOKEN_AQUI pelo token real (linha abaixo)
#
# Obtenha seu token em: https://dashboard.ngrok.com/get-started/your-authtoken
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
try:
    from google.colab import userdata
    ngrok_token = userdata.get('NGROK_TOKEN')  # Opção A: via Secrets
    if not ngrok_token:
        raise ValueError("Token vazio")
except Exception:
    ngrok_token = 'SEU_TOKEN_AQUI'  # Opção B: cole aqui o token

# Validação antes de tentar conectar
if ngrok_token == 'SEU_TOKEN_AQUI' or not ngrok_token:
    raise ValueError(
        "\n\n❌ TOKEN DO NGROK NÃO CONFIGURADO!\n"
        "Configure via Colab Secrets (Opção A) ou\n"
        "substitua SEU_TOKEN_AQUI pelo seu token real (Opção B).\n"
        "Acesse: https://dashboard.ngrok.com/get-started/your-authtoken"
    )

ngrok.set_auth_token(ngrok_token)
print("✅ Token ngrok configurado com sucesso.")

# Alternativa gratuita ao ngrok: Cloudflare Tunnel (sem limites de sessão)
# !wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -O cloudflared -q

# (leia a URL gerada no output do Colab)

url = ngrok.connect(5000)
print(f"\n✓️ Servidor ativo!")
print(f"📡 URL para colar no Studio: {url}\n")

url = ngrok.connect(5000)
url_str = url.public_url  # extrai só a URL limpa (sem 'NgrokTunnel: ...')

# Keep-alive: previne desconexão do Colab após 90 min de inatividade
import threading
def keep_alive():
    while True:
        time.sleep(25 * 60)  # ping a cada 25 minutos
        try: requests.get(url_str + '/health', timeout=5)
        except: pass
threading.Thread(target=keep_alive, daemon=True).start()

# Exibe URL uma única vez após tudo estar pronto
print(f"\n✓️ Servidor ativo!")
print(f"📡 URL para colar no Studio: {url_str}")
print(f"🕑 Keep-alive ativo — ping a cada 25 min\n")

app.run(port=5000)

Solução de Problemas

Os erros mais comuns e como resolvê-los.

!
ERR_NGROK_105 — Token inválido ou não configurado
Acontece quando o código ainda tem SEU_TOKEN_AQUI ou o token está vazio. Solução: configure o token via Colab Secrets (Opção A do Passo 4) ou substitua SEU_TOKEN_AQUI pelo seu token real no código. Obtenha o token em dashboard.ngrok.com.
!
Authentication expired or invalid
As credenciais do Google expiraram. O processo de renovacao e feito no seu computador, nao no Colab:
terminal (seu computador)
notebooklm login
Apos fazer login no Google na janela que abrir, localize o novo storage_state.json (veja Passo 2, item 3). Depois execute novamente a celula de setup no Colab, selecione o novo arquivo e reinicie a celula do servidor.
!
Não foi possível conectar (Studio)
A célula do servidor parou de rodar. Verifique no Colab se a célula ainda está executando (deve ter um ícone de loading). Se não estiver, execute novamente. A URL do ngrok vai mudar — atualize no Studio clicando em desconectar e reconectando com a nova URL.
!
GENERATION_FAILED ao gerar relatório/podcast
O Google aplicou limite de taxa temporário. Aguarde 5-10 minutos e tente novamente. É uma limitação do servidor do Google, não do sistema.
!
Colab desconecta após inatividade
O Colab gratuito desconecta após ~90 minutos de inatividade. Para evitar, interaja com o notebook regularmente ou considere o plano Colab Pro. Ao reconectar, você precisará executar novamente as células de instalação e do servidor.
💡 Salve o notebook Colab no seu Google Drive (File → Save) para não perder o código entre sessões.
!
No notebook context (sem notebook selecionado)
Você tentou executar um comando sem selecionar um notebook antes. Sempre execute use <id> ou clique em Usar na lista de notebooks antes de fazer perguntas ou gerar conteúdo.
!
Colab desconecta após ~90 minutos
O servidor inclui um keep-alive automático que faz ping a cada 25 minutos, estendendo a sessão. Se mesmo assim desconectar, o Studio exibe "⚠ Colab pode ter desconectado" na barra de status. Clique em desconectar, reinicie a célula do servidor no Colab (nova URL será gerada) e reconecte.
💡 A barra de status no topo do Studio mostra um ponto verde pulsando enquanto o Colab está ativo, e o horário do último ping. Se o ponto ficar cinza, o Colab provavelmente desconectou.
!
Studio mostra "⚠ Credenciais expiradas"
O Studio detecta automaticamente quando as credenciais do Google expiraram e exibe um alerta vermelho com instruções passo a passo. Clique no alerta para ver o procedimento completo de renovação (é o mesmo do Passo 2 deste guia).
!
Conflito ao enviar dois comandos rápido
O Studio serializa todos os comandos em uma fila interna. O badge "N na fila" ao lado do título mostra quantos estão aguardando. Envie quantos quiser em sequência — serão executados um a um sem conflito.
💡 Use o painel de histórico para exportar/importar comandos em JSON ou sincronizar com Google Drive (botão ☁). Para instalar o Studio como app nativo no computador ou celular, use o botão 📲 na barra superior. Fluxo avançado — transferindo contexto entre conversas

Uma das maiores vantagens desta integração é poder iniciar uma nova conversa com o Claude já carregada com todo o estado do projeto — notebooks, fontes, relatórios — sem precisar explicar nada. Este fluxo foi testado e validado em produção.

✓️ Exemplo validado: Em um projeto com mais de 20 fontes (documentos, relatórios, imagens), o contexto completo foi transferido para um novo agente em menos de 1 minuto. O agente recebeu o projeto, identificou os pontos principais e ficou pronto para trabalhar — sem nenhuma explicação adicional.
1
Selecione o notebook e verifique as fontes
No Studio, execute os comandos para garantir que o notebook correto está ativo e as fontes estão prontas:
studio
list --json           # veja seus notebooks e IDs
use <id>              # selecione o notebook ativo
source list --json    # confirme que as fontes estão "ready"
2
Gere um relatório (opcional mas recomendado)
Se quiser levar uma análise completa do notebook para a nova conversa, gere um relatório antes. O Studio aguarda automaticamente e faz o download quando pronto:
studio
generate report --format briefing-doc
O Studio vai mostrar o progresso em tempo real com cronômetro, executar o artifact wait automaticamente e baixar o arquivo relatorio.md assim que ficar pronto.
💡 Para relatórios longos (casos com muitas fontes), o processo pode levar 5-15 minutos. Você pode continuar usando o Studio enquanto aguarda.
3
Gere o bloco de contexto com um clique
No Studio, clique no botão Contexto na barra superior. Ele monta automaticamente um bloco limpo e estruturado, focado no notebook ativo — sem ruído de comandos operacionais.
O que o botão coleta
1 Dados do notebook ativo — nome, ID e data de criação, extraídos diretamente do servidor
2 Lista de fontes — apenas os nomes dos documentos, sem status técnico ou colunas de tabela
3 Respostas do NotebookLM — somente os resultados dos comandos ask executados na sessão, com a pergunta e a resposta limpa. Comandos operacionais (use, list, status, source list) são automaticamente ignorados
📌 A qualidade do contexto depende das perguntas que você fizer. Antes de clicar em Contexto, execute os ask mais importantes — sobre fatos comprovados, contradições, argumentos centrais. Cada resposta do NotebookLM entra no bloco e enriquece o que o Claude recebe.
Exemplo do bloco gerado:
# Contexto do Projeto — NotebookLM + Claude
Gerado em: 01/03/2026, 14:32

## Notebook ativo
- Nome: Análise de Contrato — Projeto X
- ID: a1b2c3d4...
- Fontes: 8 documentos carregados

## Fontes carregadas
  1. Contrato Principal.pdf
  2. Aditivo 01.pdf
  3. Parecer Jurídico.docx
  4. Jurisprudência Aplicável.txt
  ...

## Respostas do NotebookLM nesta sessão

Pergunta: Quais cláusulas apresentam risco para a parte contratante?
Com base nas fontes 1 e 3, as cláusulas 7.2 e 12.1 apresentam risco, pois estabelecem condições unilaterais que podem ser interpretadas como...

Pergunta: O parecer jurídico contradiz alguma cláusula do contrato?
Sim. O parecer (fonte 3) aponta que a cláusula 12.1 contraria o entendimento consolidado nos precedentes citados nas fontes 2 e 4, pois...
⚠️ O botão Contexto fica desabilitado se as credenciais do Google expirarem — o Studio detecta isso automaticamente ao conectar e ao gerar o contexto. Renove o storage_state.json antes de continuar.
4
Abra uma nova conversa com o Claude
Acesse claude.ai e inicie uma nova conversa. Cole o bloco de contexto copiado e, se gerou um relatório, anexe o arquivo .md diretamente na mensagem. O Claude consegue ler o arquivo completo de uma vez.
⚠️ Não precisa explicar o projeto — o contexto já diz tudo. Vá direto ao que precisa fazer.
5
Trabalhe com o Claude + mantenha o Studio aberto
Com o contexto carregado, o Claude pode redigir peças, estruturar teses, analisar argumentos e preparar perguntas para o NotebookLM. Mantenha o Studio aberto em paralelo para executar consultas e trazer novos dados:
🔄 Ciclo de trabalho recomendado
A Claude pede: "Execute no Studio: ask "quais documentos comprovam a tese principal?""
B Você roda no Studio → clica 📋 copiar no bloco de resultado
C Cola o resultado na conversa → Claude incorpora e constrói em cima
D Repita para cada ponto que precisar aprofundar
💡 Para relatórios longos, use sempre o botão ⬇️ Baixar do Studio e anexe o .md na conversa. O Claude lê o arquivo inteiro de uma vez, sem limite de texto colado.
🔍 Dicas rápidas do Studio: use a busca na paleta (campo no topo da coluna esquerda) para filtrar comandos rápido; o botão ⇄ notebooks troca de projeto sem digitar ID; o histórico fica salvo mesmo após fechar o Studio; e uma notificação aparece automaticamente quando a geração terminar. Comandos entram em fila automática (badge no topo). Exporte/importe o histórico ou sincronize via Drive pelo painel de histórico. Para instalar o Studio como app nativo, use o botão 📲 na topbar.
6
Resultado esperado pelo novo agente
Com o contexto transferido corretamente, o novo Claude responde algo assim:
"Contexto recebido e compreendido. Estou a par do projeto e do notebook ativo no NotebookLM. O notebook está carregado com todas as fontes disponíveis, cobrindo as diferentes frentes do tema. Posso analisar os documentos, estruturar argumentos, redigir conteúdo ou preparar perguntas para rodar no Studio. Como quer prosseguir?"
Isso significa que o agente está posicionado e pode redigir peças, estruturar teses, analisar argumentos e processar novos dados do notebook — tudo sem nenhuma explicação adicional da sua parte.
💡 Quando criar uma nova conversa vs. continuar na mesma? Crie uma nova conversa quando: (1) a conversa atual ficou longa e o Claude está perdendo contexto das mensagens antigas; (2) você vai trabalhar em um notebook diferente; (3) quer começar uma tarefa completamente nova (ex: mudar de análise para redigir uma peça). Continue na mesma conversa quando está aprofundando o mesmo ponto e o contexto acumulado é valioso.

Playbooks

Sequências de comandos prontas para os casos de uso mais comuns. Execute cada comando separadamente no Studio — um de cada vez — e em seguida use o resultado no Claude.

⚠️ Importante: cada linha é um comando independente — use o botão copiar de cada linha para colá-la no Studio individualmente. Execute um, aguarde o resultado, depois execute o próximo. Se salvar como Playbook, o Studio faz isso automaticamente pela fila.
1
Redigir uma peça jurídica
Use quando precisar redigir petições, manifestações ou memoriais com base nos documentos do notebook.
1 use <id-do-notebook>
2 generate report --format briefing-doc
# aguarda download automático do relatorio.md
Depois, no Claude: anexe o relatorio.md e diga "Com base neste relatorio, redija uma petição inicial contestando os calculos do exequente."
💡 Salve esse fluxo como Playbook no Studio. Abra o painel de Histórico, clique em Playbook, selecione os dois comandos e dê um nome. O Studio os executará um a um pela fila automaticamente.
2
Montar um dossie de provas
Ideal para mapear quais documentos comprovam cada fato relevante, com citações rastreavéis.
1 use <id-do-notebook>
# passo 1 — selecionar o notebook
2 ask "Quais fatos estão comprovados pelos documentos? Cite as fontes."
# passo 2 — aguarde a resposta e use Copiar Contexto
3 ask "Quais documentos contradizem a versao da parte contraria?"
# passo 3 — aguarde e use Copiar Contexto novamente
4 generate report --format study-guide
# passo 4 — aguarda download automático do relatorio
Cole os resultados no Claude e peça: "Organize essas provas em um dossie estruturado por eixo tematico, com os numeros de citação do NotebookLM."
3
Revisão bibliográfica / pesquisa acadêmica
Para projetos com artigos, teses e relatorios — mapear argumentos, identificar lacunas e comparar posicoes.
1 source list --json
# confirme que todas as fontes estao "ready"
2 ask "Quais sao os principais argumentos de cada fonte?"
# aguarde e use Copiar Contexto
3 ask "Onde os autores divergem entre si?"
# aguarde e use Copiar Contexto novamente
4 generate report --format study-guide
# aguarda download automático
Transfira o contexto para o Claude (botão copiar contexto) e peça: "Com base na revisao do NotebookLM, escreva um referencial teorico de 3 paginas comparando as perspectivas X e Y."
4
Análise de dados financeiros / relatorios empresariais
Para due diligence, analise de balancete, comparativo de indicadores entre periodos.
1 ask "Quais indicadores aparecem em todos os relatorios?"
2 ask "Houve queda ou crescimento ano a ano? Cite os valores exatos."
3 generate report --format briefing-doc
Cole no Claude: "Com os dados acima, redija um parecer executivo de 1 pagina apontando os 3 pontos de atencao criticos."
💡 Para este fluxo, o formato briefing-doc funciona melhor que study-guide — gera um resumo executivo com os dados-chave ja destacados.
🎯 Dica de produtividade — Playbooks: Salve qualquer sequência como Playbook no Studio. Abra o painel de histórico, clique em Playbook, selecione os comandos e dê um nome. Na próxima sessão, um clique na paleta executa tudo na ordem certa. Os playbooks ficam salvos no navegador (localStorage) — não no Colab. Isso significa que sobrevivem a reinicializações do Colab, trocas de URL do ngrok e fechamentos do Studio. O único caso em que se perdem é se você limpar os dados do navegador ou usar outro computador.
